Resumen por módulos

Los casos se agruparon por módulos funcionales para analizar el comportamiento del sistema desde la perspectiva del usuario final.

1. Autenticación (AUTH)
PA-AUTH-01PASS
Inicio de sesión con diferenciación de rol: se accedió correctamente al dashboard correspondiente.
PA-AUTH-02PASS
Manejo de error por credenciales inválidas: se muestra un mensaje genérico de error.
PA-AUTH-03PASS
Manejo de error por cuenta inactiva: se impide el acceso con la cuenta deshabilitada.
PA-AUTH-04FAIL
Recuperación de contraseña: el sistema solicita el correo pero no completa el envío del enlace.
PA-AUTH-05FAIL
Cambio de contraseña desde enlace de recuperación: no se confirma el flujo completo.
2. Gestión de Usuarios (USR)
PA-USR-01PASS
Listar usuarios con filtros: el Superadmin puede filtrar por rol, estado y búsqueda por nombre.
PA-USR-02PASS
Crear usuario: se permite registrar un usuario nuevo con datos válidos.
PA-USR-03FAIL
Editar usuario: no se guardan correctamente los cambios del perfil.
PA-USR-04FAIL
Activar/desactivar cuenta: la acción cambia el estado, pero produce efectos secundarios en el flujo de edición.
PA-USR-05PASS
Ver log de trazabilidad individual por usuario: se muestran registros paginados del usuario seleccionado.
3. Dashboard de Administración (ADM)
PA-ADM-01FAIL
Conteo de usuarios activos: el dashboard muestra listas en lugar de contadores correctos.
PA-ADM-02PASS
Accesos rápidos del dashboard: los botones de usuario y logs navegan correctamente.
PA-ADM-03PASS
Actividad reciente del sistema: se visualizan las acciones recientes con éxito.
4. Log de Trazabilidad (LOG)
PA-LOG-01FAIL
Registro automático de login/logout: se contabiliza login, pero no quedan registrados los eventos logout.
PA-LOG-02PASS
Registro de creación/edición/archivado de paciente: las acciones quedan registradas correctamente.
PA-LOG-03PASS
Registro de inicio de sesión de juego: se guarda el inicio de la sesión de rehabilitación.
PA-LOG-04PASS
Registro de fin de sesión de juego: el sistema guarda el cierre y el estado final.
PA-LOG-05FAIL
Registro de generación/descarga de reporte PDF: no se completa la descarga automática del documento.
PA-LOG-06PASS
Visualización del log global: se visualizan usuario, rol, acción, entidad y fecha/hora.
PA-LOG-07PASS
Exportar log de trazabilidad: se descarga el reporte con éxito.
PA-LOG-08PASS
Visualización del propio historial: el especialista visualiza solo sus acciones.
5. Dashboard del Especialista (ESP)
PA-ESP-01PASS
Lista de pacientes recientes: se muestran los pacientes asignados al especialista ordenados por actividad.
PA-ESP-02PASS
Resumen de última sesión por paciente: la información se presenta de forma correcta.
PA-ESP-03PASS
Accesos rápidos del dashboard: los botones abren las pantallas correctas.
6. Gestión de Pacientes (PAC)
PA-PAC-01PASS
Listar pacientes con búsqueda y filtro: se muestran los pacientes asignados con los filtros activos.
PA-PAC-02PASS
Crear paciente: se valida la información y se registra el paciente con éxito.
PA-PAC-03PASS
Editar paciente: se logra modificar la información del paciente.
PA-PAC-04PASS
Archivar paciente: la acción se ejecuta correctamente.
PA-PAC-05PASS

PA-PAC-06PASS
Historial de sesiones: se muestra el historial en orden cronológico.
PA-PAC-07PASS
Gráfica de evolución: se visualiza la comparación cuando existen suficientes sesiones.
7. Consentimiento Informado (CONS)
PA-CONS-01FAIL

PA-CONS-02FAIL
Registrar metadata del consentimiento: no se completa el registro de la información adicional.
PA-CONS-03FAIL
Visualizar/descargar consentimiento: no se logra la descarga del documento.
PA-CONS-04FAIL
Bloquear inicio de sesión sin consentimiento válido: se permite iniciar una sesión sin el documento firmado.
PA-CONS-05FAIL
Reemplazar/actualizar documento: no se completa la carga de una nueva versión.

PA-SES-01PASS
Selección del paciente para nueva sesión: se avanza correctamente a la configuración de la sesión.
PA-SES-02PASS

PA-SES-04FAIL

PA-SES-06PASS

PA-SES-07PASS

PA-SES-08PASS

PA-RES-02PASS

PA-RES-03PASS

PA-RES-05PASS

PA-HIST-02PASS

PA-HIST-03PASS

PA-REP-02PASS
Exportar/descargar reporte PDF: el archivo se descarga con éxito.

PA-CTA-01FAIL

PA-CTA-02PASS
Cambiar contraseña: la operación se completa correctamente y redirige al dashboard.
PA-CTA-03PASS

✓

Conclusión

Las pruebas de aceptación confirmaron que la mayoría de los flujos críticos del sistema funcionan desde la perspectiva del usuario, especialmente aquellos relacionados con autenticación, gestión de pacientes, sesiones terapéuticas y trazabilidad. Sin embargo, se identificaron mejoras pendientes en recuperación de contraseña, consentimiento informado, plantillas de sesión y edición de perfil, que requieren atención antes de cerrar la liberación del sistema.
