Contexto y objetivo

Proyecto: gabrielaborda_PS-ProyectoFinal-A8-Backend Integración: SonarCloud en el pipeline CI/CD Fecha: 2026-07-19

Se integró el análisis de calidad de código de SonarCloud al pipeline de GitHub Actions del backend para que el dashboard muestre métricas reales de cobertura y calidad, en lugar de advertencias de ruta y archivos no resueltos.

Diagnóstico

El análisis de SonarCloud era exitoso, pero la cobertura no se computaba porque el scanner no podía resolver las rutas absolutas que generaba coverage.py en el runner de GitHub Actions. 

WARN  Invalid directory path in 'source' element: /home/runner/work/PS-ProyectoFinal-A8-Backend/PS-ProyectoFinal-A8-Backend/app
ERROR Cannot resolve the file path '__init__.py' of the coverage report, the file does not exist in all 'source'.
ERROR Cannot resolve 64 file paths, ignoring coverage measures for those files

La causa raíz fue que coverage.xml se generaba con rutas absolutas del contenedor del runner, mientras que el scanner de SonarCloud corre en Docker y solo puede resolver rutas relativas dentro del workspace.

Solución aplicada

Se corrigió la generación de cobertura y se ajustó la configuración del proyecto y del workflow para que SonarCloud lea el reporte correctamente.

1. .coveragerc

[run]
relative_files = True
source = app

2. sonar-project.properties

sonar.projectKey=gabrielaborda_PS-ProyectoFinal-A8-Backend
sonar.organization=gabrielaborda
sonar.projectName=PS-ProyectoFinal-A8-Backend
sonar.sources=app
sonar.tests=tests
sonar.python.version=3.12
sonar.python.coverage.reportPaths=coverage.xml
sonar.sourceEncoding=UTF-8
sonar.exclusions=**/__pycache__/**,**/migrations/**,**/*.png,**/*.jpg,**/*.jpeg,**/*.svg,**/*.ico,**/assets/**

3. Workflow GitHub Actions

- name: Clonar repositorio
  uses: actions/checkout@v4
  with:
    fetch-depth: 0

Warnings resueltos

Se resolvieron los tres warnings que impedían una integración limpia de SonarCloud.

Warning Causa Solución
"code is analyzed as compatible with all Python 3 versions" No se especificó la versión de Python para Sonar Agregar sonar.python.version=3.12 en sonar-project.properties
"problems with file encoding in the source code" Sonar intentaba analizar una imagen binaria como código fuente Excluir imágenes/assets con sonar.exclusions
"Shallow clone detected" El checkout solo traía el último commit Usar fetch-depth: 0 en el paso de checkout

Conclusión

La integración de SonarCloud quedó completa para el backend, con métricas de cobertura visibles y análisis de calidad activos en cada push y pull request. El dashboard ahora lee correctamente el reporte de cobertura y ya no falla por rutas absolutas ni archivos no resueltos.

Ver dashboard en SonarCloud: https://sonarcloud.io/dashboard?id=gabrielaborda_PS-ProyectoFinal-A8-Backend&branch=main
