1. Introducción

Las pruebas de integración tuvieron como propósito verificar la correcta comunicación entre la aplicación frontend y el backend del proyecto RehaACV, asegurando que el intercambio de información mediante la API REST se realice de forma segura, consistente y conforme a los requisitos funcionales del sistema.

El backend fue desarrollado utilizando FastAPI, exponiendo servicios REST versionados bajo el prefijo /api/v1, mientras que el frontend fue implementado en React utilizando una arquitectura centralizada basada en Axios, Zustand y TypeScript para el consumo de los servicios.

2. Objetivos

  • Validar la correcta comunicación HTTP entre frontend y backend.
  • Verificar el funcionamiento de la autenticación basada en JWT.
  • Comprobar la correcta configuración de CORS.
  • Validar el manejo centralizado de errores HTTP.
  • Verificar el cumplimiento de los contratos de datos definidos mediante modelos Pydantic.
  • Confirmar el funcionamiento de los principales módulos consumidos por el frontend.
  • Comprobar la disponibilidad del backend mediante el endpoint Health Check.

3. Alcance

Se incluyen los módulos de autenticación, usuarios y perfiles, dashboard, pacientes, documentos, plantillas de sesión, sesiones clínicas, validación de contratos de datos, manejo de errores HTTP, configuración CORS y Health Check.

4. Arquitectura de Integración

ComponenteDetalle
BackendFastAPI con servicios bajo /api/v1, validación automática con Pydantic, middleware CORS y respuestas JSON tipadas.
FrontendReact + Axios + Zustand + TypeScript, con base URL configurable y interceptores de petición/respuesta.

  • El frontend consuma correctamente todos los endpoints de la API.

Conclusiones

Las pruebas de integración permitieron validar que la comunicación entre el frontend y el backend del sistema RehaACV funciona de manera correcta mediante el consumo de los servicios REST desarrollados con FastAPI. La utilización de Axios como cliente HTTP centralizado, el uso de interceptores para la administración automática del token JWT, la gestión del estado mediante Zustand y la validación de datos con Pydantic proporcionan una arquitectura robusta que reduce errores de comunicación y garantiza la consistencia de la información intercambiada.
