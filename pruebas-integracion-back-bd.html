Fixtures de configuración

Definidas en conftest.py, dan soporte transversal a toda la suite.

db

Configura la sesión de base de datos por caso de prueba, envolviendo la ejecución en transacciones para aislamiento y reversión.

client

Instancia de TestClient de FastAPI configurada con la base de datos de integración y overrides de dependencias.

admin_headers / specialist_headers

Encabezados HTTP con token JWT Bearer pre-calculado para credenciales por defecto de admin y especialista.

monkeypatch

Simula variables de configuración dinámicas y evita el envío real de correos SMTP en recuperación de contraseñas.

Conclusión técnica final

La suite de pruebas de integración desarrollada para el backend FastAPI cumple de forma rigurosa con los objetivos y criterios del plan de pruebas:

  • 01
    Robustez y veracidad Al no apoyarse en mocks para la base de datos, las pruebas aseguran que las sentencias SQL generadas por SQLAlchemy son plenamente compatibles con SQLite y que todas las restricciones de integridad y triggers se aplican en la vida real.
  • 02
    Seguridad y aislamiento Las verificaciones cruzadas entre Administradores y Especialistas confirman la efectividad de las directivas de seguridad basadas en roles, asegurando la privacidad de los datos demográficos e históricos de los pacientes.
  • 03
    Mantenibilidad a largo plazo La parametrización de credenciales fijas y la configuración estructurada de base de datos temporal facilitan la ejecución automática en pipelines de CI/CD, evitando regresiones lógicas ante futuros cambios del backend.
